# Contributing to Genotype

## Code Comments

### References

The `// SEE:` comments followed by an ID in the format `![\W_ID]` are comment references. These allow linking to common codebase gotchas and patterns that might not be obvious or unorthodox. See the [Code Comments References](./docs/contributing/comments.md#references) for more details about a specific reference.

Use this pattern to link to repeating explanations without making the code comments too verbose. Don't abuse this pattern, and only use it for non-trivial explanations that are repeated in multiple places or when an extra detailed explanation is required.
